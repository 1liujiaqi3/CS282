Program Description

This assignment involves writing an Android program that has the following features:

  • It contains an MapAppActivity class that uses a TextView object to prompt for the location and then stores the latitude and longitude in two EditText objects.

  • It uses a Button object with the label "Show Location" to run the showLocation() method that uses the lat/long values provided by the user to start an activity that displays a map of the location.

  • The Button that shows the location must be connected to the MapAppActivity.showLocation() method via the android:onClick="showLocation" attribute.

  • Make sure to carefully check for errors in the latitude and longitude values entered by the user. If there are any problems with the input please display helpful error messages via Android Toasts.

  • The activity that displays the map must be started via an intent and must not be written by you, e.g., the Browser app will handle URI's with the prefix http://maps.google.com.
The entire layout of the application must be structured by the contents of the Android resource files described via XML, i.e., you must not hard-code any TextView, EditText, or Button objects into your MapAppActivity class.

All the lifecycle events associated with your MapAppActivity class must be logged to LogCat via the Log.d() method, e.g.:

Additional Work for Graduate Students

Graduate students (or students who are taking this course for graduate credit) must do the following additional work:
  • You must support default latitude and longitude values if the user doesn't provide any. By default, these values should point to Vanderbilt University.

  • First try to use the geo: URI intent solution, which should work if you're running on an Android phone (I recommend using the approach described here to see if this works). If the geo: URI intent doesn't work (e.g., if there is no app configured to handle it) then use the http://maps.google.com URI intent instead, which launches the Browser app.

Concluding Remarks

The MapApp is intentionally designed to be easy to help you get familiar with installing/using Eclipse and Android. Information useful for learning how to write this application is available in Week 2 of the MOOC Programming Mobile Applications for Android Handheld Systems, which requires you to register for this MOOC. Likewise, the the Busy Coder's Guide to Android Development textbook. I recommend you read this material. It's also helpful to work through the various tutorials in that book.

If you find it extremely hard to complete this assignment you may not be ready to take this class. If this is the case, I recommend you take the Intermediate Software Design course (CS 251) first since it focuses on the patterns and tactics of effective OO design and programming.
